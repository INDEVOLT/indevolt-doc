# Modbus TCP 说明

## 1. 什么是 Modbus TCP？

Modbus 是一种常见的设备通信协议，用于让不同设备之间交换数据。

Modbus TCP 是基于家庭**局域网**（LAN）进行通信的 Modbus 方式。设备通过路由器、交换机或网线连接后，即可在本地网络中互相通信。

可以把它想象成“问答”过程：
1. 系统发出请求： “现在电池电量是多少？”
2. 储能设备接收到请求
3. 设备返回数据：“电量是 65%”

通过 Modbus TCP，你可以：

- 接入家庭能源管理系统（HEMS），实现自动化控制
- 本地监控设备运行状态，不依赖云平台
- 实时读取 SOC、电压、功率、温度等数据
- 与光伏、电表、充电桩等设备联动

## 2. 系统架构

```text
┌──────────────┐
│   HEMS / PC  │
└──────┬───────┘
       │ 局域网（LAN）
       │
┌──────▼───────┐
│   路由器      │
└──────┬───────┘
       │
┌──────▼───────┐
│   微储        │
└──────────────┘
```

| 设备      | 角色          | 说明                   |
| --------- | ------------- | ---------------------- |
| HEMS / PC | Modbus Client | 主动发起读取或写入请求 |
| 微储      | Modbus Server | 响应请求并返回数据     |

## 4. 如何使用

### 4.1 准备

开始前，请确认以下内容：

- ✅ 设备支持 Modbus TCP 功能
- ✅ 设备已经正常联网
- ✅ 电脑 / HEMS / 第三方系统与设备处于同一局域网


### 4.2 开启 Modbus TCP

设备的 Modbus TCP 功能默认关闭，需要在 App 中手动开启。



### 4.3 配置连接参数

在第三方系统或 Modbus 工具中，填写以下参数：

| 参数     | 说明                       |
| -------- | -------------------------- |
| 设备 IP  | 微储的 IP 地址             |
| TCP 端口 | 默认为`8899`                     |
| Slave ID | 设备标识，默认为`1` |

### 4.4 推荐读取频率

建议读取周期： 秒

频繁读取可能增加设备通信负载，影响通信稳定性。

### 4.5 常用功能码

| 功能码 | 说明                                     |
| ------ | ---------------------------------------- |
| 03     | 读取保持寄存器（Read Holding Registers） |
| 04     | 读取输入寄存器（Read Input Registers）   |
| 06     | 写单个寄存器（Write Single Register）    |
| 16     | 写多个寄存器（Write Multiple Registers） |


### 4.6 操作示例

#### 步骤 1：获取设备 IP 地址

确保设备已经联网，并记录设备 IP 地址。IP 地址可以
- 在路由器后台查看
- 在 App 的设备设置 → 网络设置 中查看

#### 步骤 2：使用 Modbus 工具连接设备

在电脑中打开 Modbus TCP 工具，并填写：
- IP 地址
- 端口：8899
- Slave ID：1


#### 步骤 3：读取寄存器数据

输入对应寄存器地址后，发送请求，即可获取数据。详见下方[寄存器说明](#寄存器说明)。

## 6. FAQ

<details>
  <summary>**Q: 无法连接设备**</summary>

  请检查：
  - 设备是否已经开启 Modbus TCP
  - 设备与电脑是否在同一局域网
  - IP 地址是否正确
  - 端口是否填写为 8899
</details>

<details>
  <summary>**Q: 读取数据失败**</summary>

  请检查：
  - 网络连接是否正常
  - 功能码是否正确
  - 寄存器地址是否正确，地址是否从 0 开始计算
  - 数据类型是否匹配
</details>
